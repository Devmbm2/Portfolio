RAG Agent with Supabase, PostgreSQL, and Google Drive in n8n

In this quick tutorial, I show you how to set up PostgreSQL and Supabase credentials in n8n to build a working RAG (Retrieval-Augmented Generation) agent with memory. You'll also learn how to integrate Google Drive for document intake and update the RAG vector database automatically.

Project Objective

The goal is to create a smart, memory-aware RAG agent that can pull relevant context from your document database and respond intelligently. This includes seamless intake from Google Drive and automated vector updates.

Workflow Overview

  • Set up and connect Supabase with PostgreSQL to store vector embeddings
  • Use n8n to extract documents from Google Drive
  • Generate and update embeddings for new or updated documents
  • Store embeddings in Supabase for vector search
  • Use OpenAI to generate responses using context retrieved from the vector DB
  • Maintain memory across user sessions for coherent conversations

Key Features

  • PostgreSQL + Supabase for RAG vector storage
  • Google Drive file monitoring and ingestion
  • OpenAI integration for smart responses with context
  • Memory tracking using session IDs and message history
  • Fully no-code setup using n8n

Use Cases

  • Internal document Q&A agents
  • Customer support bots with access to company knowledge base
  • Personal research assistants with file awareness

Results

By combining Supabase, PostgreSQL, Google Drive, and OpenAI in n8n, we’ve built a responsive and memory-aware RAG agent that continuously learns from new documents—all without writing backend code.
